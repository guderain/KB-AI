# KB-AI 前后端落地方案（NotionNext + FastAPI + LangChain）

## 1. 目标
- 从 `D:/知识库` 中扫描 Markdown 文档构建向量索引。
- 提供基于 LangChain 的 RAG 问答 API。
- 在 NotionNext 前端新增 AI 问答页面并对接 FastAPI。

## 2. 技术栈
- 前端：NotionNext (Next.js)
- 后端：FastAPI + LangChain
- 向量库：Milvus
- 缓存：Redis
- 业务库：PostgreSQL

## 3. 架构
- `NotionNext`：负责 UI、用户输入、流式渲染答案和来源展示。
- `KB-AI`：负责检索、生成、索引、会话与日志落库。
- `Milvus`：存储向量与检索。
- `PostgreSQL`：存文档元信息、问答日志。
- `Redis`：热门问答缓存、会话上下文缓存。

## 4. 数据流
1. 离线索引：
   - 扫描 `D:/知识库/**/*.md`
   - 文档分块（`chunk_size=1000`, `chunk_overlap=200`）
   - 生成 embedding 并写入 Milvus
   - 同步元数据到 PostgreSQL
2. 在线问答：
   - 前端提交问题到 `POST /api/v1/chat`
   - Redis 命中直接返回
   - 未命中时：Milvus 检索 TopK -> LLM 生成答案
   - 返回答案和引用来源，并写入缓存与日志

## 5. API 设计
- `GET /api/v1/health`：健康检查
- `POST /api/v1/chat`：问答接口
- `POST /api/v1/chat/stream`：流式问答（SSE）
- `POST /api/v1/ingest/reindex`：重建索引

## 6. NotionNext 对接
1. 新增页面：`/pages/ai.tsx`（聊天 UI）。
2. 环境变量：
   - `NEXT_PUBLIC_KB_AI_BASE_URL=http://localhost:8000`
3. 前端调用：
   - 非流式：`POST /api/v1/chat`
   - 流式：`POST /api/v1/chat/stream`
4. 显示来源：把 `sources` 渲染成可点击列表。

## 7. 部署建议
1. 开发：本地 docker-compose 启 Redis/PostgreSQL/Milvus。
2. 生产：
   - FastAPI 与 NotionNext 分离部署。
   - 由 Nginx 统一反代到同域（例如 `/api/rag/*` -> FastAPI）。
   - 增加 API Key 鉴权、限流、日志审计。

## 8. 迭代路线
1. MVP：单库问答 + 来源引用 + 缓存。
2. V2：重排模型（Rerank）、对话记忆、多轮上下文优化。
3. V3：权限过滤（按用户/标签）、增量索引、监控告警。
