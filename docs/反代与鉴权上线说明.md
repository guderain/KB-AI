# 反代与鉴权上线说明

## 1. 当前已生效的后端鉴权
- 接口读取 `X-API-Key`，对比 `.env` 中 `API_KEYS`（逗号分隔）。
- 若未配置 `API_KEYS`，则鉴权关闭。
- 已启用限流：
  - `RATE_LIMIT_PER_MINUTE`（普通接口）
  - `INGEST_RATE_LIMIT_PER_MINUTE`（索引接口）

## 2. 本地已写入的 API Key
- 已写入 `KB-AI/.env` 的 `API_KEYS`。
- 你可以再追加多个 key：`API_KEYS=key1,key2,key3`

## 3. Nginx 配置
- 模板文件：`docs/nginx-notionnext-kbai.conf`
- 需要替换：
  - `server_name your-domain.com`
  - `__KB_AI_API_KEY__` -> `KB-AI/.env` 的 key

## 4. 关键安全建议
1. 生产环境请开启 HTTPS。
2. `ingest` 接口默认建议仅内网/本机访问（模板里已 `allow 127.0.0.1`）。
3. 若你希望前端不直接访问 `/api/v1/*`，可改为 Next.js 服务端中转再注入 key。
