# 腾讯云同域上线步骤（wangguanxi.space + /kb-ai）

## 1. 部署目标
- 前端：`https://wangguanxi.space` -> `127.0.0.1:3000`
- 后端：`https://wangguanxi.space/kb-ai/*` -> `127.0.0.1:8000/*`

## 2. 安装 Nginx
```bash
sudo apt update
sudo apt install -y nginx
```

## 3. 下发配置
把 `docs/nginx-notionnext-kbai.conf` 上传到服务器，并执行：
```bash
sudo cp nginx-notionnext-kbai.conf /etc/nginx/sites-available/wangguanxi.space.conf
sudo ln -sf /etc/nginx/sites-available/wangguanxi.space.conf /etc/nginx/sites-enabled/wangguanxi.space.conf
sudo rm -f /etc/nginx/sites-enabled/default
sudo nginx -t
sudo systemctl reload nginx
```

## 4. HTTPS 证书
```bash
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d wangguanxi.space -d www.wangguanxi.space
```

## 5. 前端环境变量（同域反代）
NotionNext 服务端环境变量：
```env
KB_AI_BASE_URL=https://wangguanxi.space/kb-ai
KB_AI_API_KEY=你的后端API_KEY
```

## 6. 后端环境变量（关键）
KB-AI `.env` 至少包含：
```env
KNOWLEDGE_BASE_DIR=/data/knowledge
AUTO_INGEST_ON_STARTUP=true
MILVUS_URI=http://127.0.0.1:19530
```

`AUTO_INGEST_ON_STARTUP=true` 会在后端启动时自动修复“Milvus 空库”场景。

## 7. 首次上线后立即灌库
```bash
curl -X POST 'http://127.0.0.1:8000/api/v1/ingest/reindex' \
  -H 'X-API-Key: 你的后端API_KEY'
```

## 8. 验证
```bash
curl -I https://wangguanxi.space
curl -I https://wangguanxi.space/kb-ai/api/v1/health
```

前端提问时确认：
- 浏览器请求 `/api/kb-chat`（由 Next.js 服务端代理）
- 浏览器请求头不直接携带 `X-API-Key`
